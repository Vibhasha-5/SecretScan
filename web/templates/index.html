Methodology

How It Works

Two complementary detection engines that cover known patterns and unknown secrets alike.

01
Pattern Detection
Curated regex patterns for 30+ secret types across AWS, GitHub, Stripe, OpenAI, Slack, Google, Azure, Twilio, and more. Each detector follows the Single Responsibility Principle — extensible and auditable.
02
Shannon Entropy Analysis
Computes H = −Σ p(x) log₂(p(x)) for every string in the file. Strings above threshold 4.5 in base64 or hex character sets are flagged — catching secrets that don't match any known pattern.
03
Severity Classification
Every finding is classified CRITICAL, HIGH, MEDIUM, or LOW based on potential blast radius. CRITICAL findings — AWS keys, database credentials, private keys — fail CI pipelines with exit code 1.
04
Safe Reporting
Matched secrets are masked in output (sk_l****p7dc) so reports can be logged and shared safely. JSON output integrates with SIEM tools, dashboards, and Slack notifications.

Pipeline Integration

CI/CD Setup

Block secrets from reaching production. SecretScan returns exit code 1 on CRITICAL or HIGH findings — your pipeline fails automatically.

1
Add SecretScan to your repo
Copy the secret-api-key-detector/ folder into your project root. Install dependencies with pip install -r requirements.txt.
2
Choose your platform
Select GitHub Actions, GitLab CI, Jenkins, or Bitbucket from the templates on the right. Copy the YAML into your repo.
3
Commit the workflow file
Push to your repo. The scan runs on every push and pull request. Merges are blocked when secrets are found.
4
Review JSON artifacts
Reports are saved as pipeline artifacts with timestamps. Download from your CI dashboard or pipe into a SIEM tool.
5
Add a pre-commit hook
Run python cli.py scan . before every commit to catch leaks at the developer workstation — before they're ever pushed.
Workflow Template
# .github/workflows/secret-scan.yml
name: Secret Detection
on:
  push: { branches: [main, develop] }
  pull_request: { branches: [main] }
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.11' }
      - run: pip install -r requirements.txt
      - run: |
          python cli.py scan . \
            --format json \
            --output scan.json
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: scan-report
          path: scan.json
# .gitlab-ci.yml
secret-scan:
  stage: test
  image: python:3.11-slim
  script:
    - pip install -r requirements.txt
    - python cli.py scan . --format json
  artifacts:
    when: always
    paths: [output/reports/*.json]
    expire_in: 30 days
  only: [main, merge_requests]
// Jenkinsfile
pipeline {
  agent any
  stages {
    stage('Secret Scan') {
      steps {
        sh '''
          pip install -r requirements.txt
          python cli.py scan . \
            --format json
        '''
      }
    }
  }
}
# bitbucket-pipelines.yml
pipelines:
  default:
    - step:
        name: Secret Detection
        image: python:3.11-slim
        script:
          - pip install -r requirements.txt
          - python cli.py scan . --format json

Test a Config Before Committing

Paste a .env or CI config below and scan it instantly.

Never commit .env filesAdd .env to .gitignore. Ship a .env.example with placeholder values.
Use a secrets managerAWS Secrets Manager, HashiCorp Vault, or GitHub Secrets. Never pass keys as plaintext environment variables in CI logs.
Rotate immediately on exposureIf a secret was ever committed — even briefly — assume it is compromised. Revoke and regenerate before doing anything else.
Audit git historySecrets removed in a later commit are still in history. Use git log -p or truffleHog to scan historical commits.

Shannon Entropy
Detects secrets outside known patterns by computing H = −Σ p(x)log₂(p(x)). Base64 and hex strings above threshold 4.5 are flagged as high-entropy candidates regardless of their content.
Modular Detectors
Three single-responsibility detector classes — API keys, auth tokens, and generic secrets — each extending BaseDetector. Clean SOLID architecture makes adding new patterns a one-file change.
Exit Code Gate
Returns exit code 1 on CRITICAL or HIGH findings, making it a drop-in CI/CD quality gate. JSON reports integrate with SIEM tools, Splunk, Grafana, and custom alerting pipelines.
